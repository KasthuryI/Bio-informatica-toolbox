ABOUT

What is TrimTech?

TrimTech is a website you can use to perform quality controls on DNA-sequences. It does so by analysing a FastQ file. TrimTech visualizes these quality controls for you. You can decide whether or not you want to trim your DNA-sequence to increase its quality. If you have chosen to trim your DNA-sequence TrimTech will visually compare your sequence before and after trimming it.


What is a FastQ-file?

If you want to know more about how this process works it’s important to know what a FastQ-file is.
A FASTQ format is a text file format that stores:

  • base sequences
  • confidence for each base in a sequence
  • describing the origin of the read down to its position on the sequencing platform flow-cell
A FASTQ file is an extension of the FASTA file format, with the extension allowing for the storage of sequencing quality data along with the sequence itself and the sequence ID. All FASTQ files consist of a set of reads, with each read having 4 lines of data. The first line of each set will always begin with an “@” symbol and is often called the sequence identifier/metadata line. The “@” is followed by the sequence identifier and an optional description. The second line will always contain the raw nucleotide sequence, but if a nucleotide base is undefined it will show an N. The third line is a spacer that will start with a “+”. There may be additional comments put on this line, such as the same content as the first line, but important data will not be found in this line. The fourth line contains the quality score of the nucleotide bases based on an ASCII table.


What is a PHRED-score?

A Phred score, also known as a Phred quality score/Q score, defines the probability that the base call generated by automated DNA sequencing is correct. It was originally developed for a computer program called Phred to help in the automation of DNA sequencing in the Human Genome Project. Phred quality scores are assigned to each nucleotide base call-in automated sequencer traces. The FASTQ format encodes phred scores as ASCII characters along the read sequences. The quality of the scores ranges from 4 to 40 in most useable phred-scaled base quality scores. However, phred-scaled quality scores in general can range from 0 to infinity. Phred scores are important in determining whether a sequence is accurate and good quality. A higher score indicates a higher probability that a sequence is correct, while a lower score indicates a higher probability that a sequence is incorrect.

  • Q: The Q stands for the Phred quality score/Q score
  • P: The P stands for the base-calling error probabilities

ASCII

ASCII has been the standard for encoding characters to numbers since well before modern DNA sequencing were developed. This encoding scheme was initially developed for a machine known as the teletype, where the user could type a message while connected to a phone line, and the message would be printed elsewhere. Because the original computer keyboards were teletype devices connected to early mainframes, this standard has persisted in computing for decades and was an easy choice for encoding a one- or two-digit value to a single character.



Phred – 33 and – 64

To encode a quality score as a single character, it will need to be compressed first, or encoded, down to a two-digit number by Phred encoding. For a call with a confidence of 99.9%, we would get a Phred score of 30. In ASCII is 30 the backspace character, which does not print, or could even delete the character before it. To get around it, the character encoding scheme adds in a base value. The most used base value is 33, which corresponds to the first visible printing character: the exclamation point. A less commonly used standard, but one that was often used with Phred encoding is 64. This value was chosen because scoring schemes will often not have 0 as a possible quality score, meaning that the @ symbol would not be a potential character for quality encoding and could be left to exclusively signify the beginning of a FASTQ sequence ID line. The common method of denoting the encoding scheme Is to first list the scheme for scaling numbers and then the signed base value. The most used scheme, and the current standard of this writing, is Phred – 33.
